컴퓨터를 이용한 코딩 방법에 대하여 설명합니다.

1. 사전작업
아래 두 가지 방법 모두 ADB(Android Debug Bridge)를 이용한 방법으로, ADB 연결을 이용하기 위해서는 아래의 사전작업을 수행하여 USB 디버깅을 활성화 해야 합니다.

아래의 휴대폰 메뉴는 제조사마다 다를 수 있습니다.

  • 설정 - 휴대폰 정보 메뉴에서 빌드 번호를 7번 터치하면 개발자가 되었다는 메세지가 발생합니다.


  • 설정 - 일반 메뉴에 "개발자 옵션" 메뉴가 생성되었습니다.


  • 개발자 옵션에서 USB 디버깅을 활성화 합니다.




2. SCRCPY를 이용한 핸드폰 원격제어
scrcpy는 genymobile에서 배포하는 무료 핸드폰 원격제어 툴 입니다.
scrcpy는 adb 연결을 이용하여 핸드폰을 제어하는 원리이며, 휴대폰에 별도의 앱 설치 없이 간단한 실행만으로 원격제어를 할 수 있습니다.

  • 윈도우 사용자
https://github.com/Genymobile/scrcpy 페이지에서,
https://github.com/Genymobile/scrcpy/releases/download/v1.12.1/scrcpy-win64-v1.12.1.zip 파일을 다운로드 후 압축을 해제합니다.

  • 리눅스(우분투) 사용자
apt install scrcpy


설치 완료 후, 휴대폰을 컴퓨터에 연결하고, scrcpy를 실행시킵니다.
최초 adb 연결을 하는 경우, 핸드폰 화면에 승인하겠냐는 메세지가 뜨며, 항상 허용을 체크하고 확인을 눌러줍니다.

그러면 휴대폰 화면이 컴퓨터에 나타납니다.
키보드를 이용하여 바로 타이핑 하시면 됩니다.
참고로 한영 전환키는 쉬프트+스페이스 입니다.





3. ADB를 통한 파일 전송
컴퓨터에서 사용 중인 IDE(Integrated Development Environment)를 이용하고자 하는 경우,
ADB를 통해 휴대폰의 파일을 복사했다가 다시 옮겨넣는 방법으로 사용할 수 있습니다.

VSCode, 브위님의 봇 시뮬레이터 등 원하는 IDE를 선택할 수 있어 조금 더 편한 환경을 구축할 수 있는 방법입니다.

ADB의 설치는 https://developer.android.com/studio/releases/platform-tools 구글의 공식 배포도구를 다운로드 받으셔도 되고, 위 1에서 언급한 scrcpy의 압축파일에도 adb가 숨어있습니다.

ADB에는 여러 기능들이 있지만, 우리가 사용할 것은 shell,pull, push입니다.

먼저 휴대폰에 있는 파일을 꺼내봅시다.
※ 메봇R 기준으로 설명합니다.
  • adb shell ls /sdcard/봇폴더/Bots/봇이름     : 소스파일 이름 확인(index.js 또는 봇이름.js)
  • adb pull /sdcard/봇폴더/Bots/봇이름/봇소스.js .    : 한칸 띄우고 .을 붙이며, 현재 디렉토리에 저장한다는 뜻입니다.
pull을 완료하면 현재 디렉토리에 저장되며, 원하는 IDE로 해당 파일을 불러와서 작업을 할 수 있습니다.

작업 완료 후에는 push를 이용해서 핸드폰에 넣어줍니다.
  • adb push 봇소스.js /sdcard/봇폴더/Bots/봇이름/.

이후, 카카오톡 봇 앱에서 해당 봇의 소스를 보면 변경된 소스가 로드된 것을 확인할 수 있습니다.

위 두가지 방법을 혼합하여,
PC에서 코딩 후, adb를 이용하여 변경된 파일을 전송하고, scrcpy로 원격제어하여 새로운 봇 소스를 테스트하면 따로 휴대폰을 만지지 않고도 PC에서 모든 작업을 수행할 수 있습니다.
