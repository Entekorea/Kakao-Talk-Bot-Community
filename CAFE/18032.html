제발 (수정하든 그냥 배포하든)재배포 할때는 저작권자 표기좀 지킵시다......

선착순 모집합니다.

(다른언어로도 할건 많은데 자바스크립트만 올인할순 없기에......)

아래 채팅방으로 들어오세요.

https://open.kakao.com/o/g4EWtFac

채팅방 입장감지와 좋아요 반응봇

버전 : 1.1.0.0

-1.0.0.0 : 최초개발 및 배포

-1.0.1.0 : 미발송 버그 수정(첫 가동시 한번이라도 채팅발송)

-1.1.0.0 : 쓰레드 중복실행 방지 기능 추가+봇 가동시 토스트 메시지 알림 기능 추가(2020년 5월 14일)

설명 : 채팅방에 새로 들어오거나 좋아요를 눌러주는경우 반응을 해줍니다.(루팅하고 상관없이 동작)

저작권자

-kmb패밀리( http://kmbfamily.net )

-너의톡방은( http://yourtalk.net )

원 저작권자

-KeonWoo PARK( http://develope.kr )


본 소스코드는 CC BY-NC-SA 라이선스를 따릅니다.

저작권자의 허락 없이 상업적 사용, 2차 무단 배포 및 무단 수정을 금합니다.

단 비상업적인 목적으로만 수정 및 사용이 가능하며

이 경우에도 저작권자는 표기하여야 합니다.

본 소스코드에 대해 궁금하신점 또는 오류가 있으시면

kmb패밀리 이메일 webmaster@kmbfamily.net

또는

너의톡방은의 문의방 https://open.kakao.com/o/gNDClgTb 으로

문의주세요.


사용법

-아무한테나 채팅이 하나라도 오면 Log를 남기고 봇이 가동됩니다.(가동 되었는지 안되었는지,잘 돌아가는지 (뻥)터졌는지 확인은 로그 읽어보세요.)

-메신저봇 기준 맨 윗줄 const scriptName 이 선언된 다음줄인 2번째줄 부터 붙여넣기 해주시면 됩니다.

-const scriptName(이 봇 소스코드를 돌리는 스크립트의 이름)이 반드시 선언되어 있어야 합니다.

-소스코드 수정과 같이 껏다 재가동 해야 하는 경우 끄고 나서(여기서 끈다의 의미는 Api.off(scriptName); 함수로 끄는것과 같은방식을 의미) 1초후에 켜야 합니다.

안그러면 종료되지 않은 쓰레드 때문에 도배될수도 있습니다.

-변경 및 설정해야 하는 변수는 5개 입니다.

g_data["interval"] , g_data["room"]["link"] , g_data["room"]["title"] , g_data["room"]["likeup"] , g_data["room"]["usersup"]

변수설명

  • g_data["interval"] : 채팅방 정보를 불러오는 간격입니다.(단위 초) 짧으면 짧을수록 금방 반응하고 길면 늦게 반응합니다.
  • g_data["room"]["link"] : 채팅방 링크 영역 입니다.(배열로 넣습니다.) 채팅방 링크가 https://open.kakao.com/o/ninemuses 면 open.kakao.com/o/ 뒤에 ninemusis 만 넣어주세요.
  • g_data["room"]["title"] : 채팅방 제목을 넣어주세요.(배열로 넣습니다.) 위 링크순서에 맞게 넣어주셔야 하고 제목값 잘못넣으면 채팅방 인원이 늘거나 좋아요 눌러준방에 톡 발송이 안되거나 엉뚱한방에 날아갑니다.
  • g_data["room"]["likeup"] : (누군가)좋아요 눌렀을때 채팅방에 보낼 내용을 적어주세요.
  • g_data["room"]["usersup"] : (누군가)새로 들어와서 방 인원이 늘었을때 보낼 내용을 적어주세요.

발생하는 오류 해설

1.채팅방 링크 배열과 채팅방 제목 배열 개수가 서로 일치하지 않습니다.

-g_data["room"]["link"] 배열 원소 개수가 3개고 g_data["room"]["title"] 배열 원소 개수가 2개인식으로 서로 원소의 개수가 불일치 하면 발생합니다.

2.Api 호출에 실패하였습니다.

-인터넷이 연결 안되어 있거나 Api서버에서 본인의 IP주소가 블럭(차단)된 경우 또는 서버점검중인 경우 입니다.

3.채팅방 정보를 불러오는데 실패하였습니다.

-올바르지 않거나 존재하지 않는 오픈채팅방 링크를 넣었을때 발생합니다.

기타

1.api서버에 이상이 있거나 어디에 문제가 있으면 오류를 남기고 봇이 강제종료 됩니다.

원인이 뭔지는 Log 읽어보세요.

2.소스코드상에 scriptName 상수(스크립트명)가 선언되어 있어야 합니다.

메신저봇 이라면 맨 윗 첫째줄에 선언되어 있기 때문에 2번째줄부터 붙여넣기 해주시면 됩니다.


이렇게

만약 scriptName 상수가 선언되어 있지 않거나 스크립트명이 잘못된 경우 (봇에의한 도배 같은)안전을 위해 동작하지 않거나 강제종료 됩니다.

3.메신저봇 이외서의 작동에 대해서는 보장하지 않습니다.

예시(채팅방 2곳에서 운영한다고 가정하고 간격 180초)

채팅방 이름 : 나인뮤지스 팬클럽 , 카트라이더 채팅방

채팅방 링크 : https://open.kakao.com/o/ninemuses , https://open.kakao.com/o/kartrider (순서대로 왼쪽거는 나인뮤지스방, 오른쪽거는 카트라이더방)

g_data["interval"] = 180;

g_data["room"]["link"] = ["ninemuses","kartrider"];

g_data["room"]["title"] = ['나인뮤지스 팬클럽','카트라이더 채팅방'];

채팅방 이름 : 나인뮤지스 팬클럽 , 카트라이더 채팅방
채팅방 링크 : https://open.kakao.com/o/ninemuses , https://open.kakao.com/o/kartrider (순서대로 왼쪽거는 나인뮤지스방, 오른쪽거는 카트라이더방)
g_data["interval"] = 180;
g_data["room"]["link"] = ["ninemuses","kartrider"];
g_data["room"]["title"] = ['나인뮤지스 팬클럽','카트라이더 채팅방'];
*/
var trigger = false;
g_data = new Array();
g_data["open"] = false;
g_data["room"] = new Array();
//설정영역
//채팅방 상태 감지 간격
g_data["interval"] = 120;
//톡방 영역
g_data["room"]["link"] = [""];                        //방 링크 영역
g_data["room"]["title"] = [''];                        //방 제목 영역(위 링크의 방 제목)
//반응시 톡 발송내용 영역
